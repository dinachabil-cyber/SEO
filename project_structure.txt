# SEO Project Structure

## Root Level Files
- `.env`, `.env.dev`, `.env.test` - Environment configuration files
- `.editorconfig`, `.gitignore` - Configuration for editors and git
- `composer.json`, `composer.lock` - PHP dependencies (Symfony project)
- `compose.yaml`, `compose.override.yaml` - Docker configuration
- `phpunit.dist.xml` - PHPUnit testing configuration
- `symfony.lock` - Symfony flex lock file
- `importmap.php` - Asset management configuration
- `README.md` - Project documentation
- `LICENSE` - Project license
- `body-section.json` - Section configuration
- `check_page.php`, `check_page_direct.php` - Page checking scripts
- `test-entity.php`, `test-query.php` - Test scripts
- `screenshot.png`, `test-public-page.png` - Screenshots

## Directory Structure

### assets/ - Frontend Assets
- `app.js` - Main JavaScript file
- `stimulus_bootstrap.js` - Stimulus framework bootstrap
- `controllers.json` - Stimulus controllers configuration
- `styles/` - CSS styles
  - `app.css` - Main stylesheet
  - `page-builder.css` - Page builder styles
- `controllers/` - Stimulus controllers
  - `csrf_protection_controller.js` - CSRF protection controller
  - `hello_controller.js` - Example hello controller

### bin/ - Binary Scripts
- `console` - Symfony console
- `phpunit` - PHPUnit executable

### config/ - Configuration Files
- `bundles.php` - Registered Symfony bundles
- `routes.yaml` - Main routing configuration
- `services.yaml` - Dependency injection configuration
- `preload.php` - Preload configuration
- `reference.php` - Reference configuration
- `packages/` - Package-specific configurations (Doctrine, Security, Twig, etc.)
- `routes/` - Route files for specific environments

### migrations/ - Database Migrations
- `Version20260303000001.php`, `Version20260303000002.php`, `Version20260303000003.php` - Migration files

### public/ - Web Root
- `index.php` - Entry point

### src/ - Source Code
- `Kernel.php` - Symfony kernel

#### Command/
- `CreateReferenceSectionCommand.php` - Command to create reference sections

#### Controller/
- `FrontController.php` - Frontend page controller (handles '/' and '/{slug}')
- `PageBuilderController.php` - Page builder administration
- `PageController.php` - Page management
- `ReferenceSectionController.php` - Reference section management
- `SiteController.php` - Site management
- `TestController.php` - Test controller

#### Entity/
- `Page.php` - Page entity
- `PageSection.php` - Page section entity
- `ReferenceSection.php` - Reference section entity
- `Site.php` - Site entity

#### Form/
- `PageType.php`, `PageSectionType.php` - Page and section forms
- `ReferenceSectionType.php`, `SaveReferenceType.php` - Reference section forms
- `SiteType.php` - Site form

#### Repository/
- `PageRepository.php`, `PageSectionRepository.php` - Page and section repositories
- `ReferenceSectionRepository.php` - Reference section repository
- `SiteRepository.php` - Site repository

### templates/ - Twig Templates
- `base.html.twig` - Base template

#### admin/ - Admin Panel Templates
- `page/` - Page management
  - `builder.html.twig` - Page builder
  - `new.html.twig`, `edit.html.twig` - Page forms
  - `insert_reference.html.twig` - Insert reference modal
- `reference/` - Reference section management
  - `index.html.twig`, `new.html.twig`, `edit.html.twig`
- `section/` - Section management
  - `new.html.twig`, `edit.html.twig`, `save_reference.html.twig`
  - `_form.html.twig` - Section form partial
- `site/` - Site management
  - `index.html.twig`, `new.html.twig`, `edit.html.twig`, `show.html.twig`

#### front/ - Frontend Templates
- `page.html.twig` - Frontend page template
- `sections/` - Section templates
  - `_hero.html.twig`, `_header.html.twig`, `_footer.html.twig`
  - `_body.html.twig`, `_cards.html.twig`, `_cta.html.twig`
  - `_faq.html.twig`, `_form.html.twig`, `_image.html.twig`

#### test/ - Test Templates
- `index.html.twig` - Test template

### tests/ - PHPUnit Tests
- `bootstrap.php` - Test bootstrap file

### translations/ - Translation Files
- `.gitignore` - Ignore translations in git (likely dynamically generated)

### var/ - Variable Files (Generated)
- `cache/` - Cache files
- Other temporary files
